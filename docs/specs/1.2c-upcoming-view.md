# Feature 1.2c: Upcoming view (calendar)

**Category:** `dashboards/command-center` — build on scaffold from 1.2a

## Done when

- Upcoming view at `/upcoming` shows a monthly calendar grid
- Grid aggregates items from four tables with color-coded dots:
  - Actions (`due_date`) — blue=open, amber=in_progress, green=done
  - Family activities (`date` or `day_of_week`) — purple
  - Important dates (`date`) — red
  - Maintenance tasks (`next_due`) — coral
- Clicking a day opens a detail panel listing all items for that day
- Week view toggle provides a denser look-ahead
- Thoughts are deliberately excluded (commitments only)

## What to build

### Components

**`CalendarGrid.tsx`** — monthly grid:
- Standard month layout with day cells
- Each cell shows colored dot indicators by source type + item count
- Navigation: prev/next month arrows
- Toggle between month view and week view
- Query all four tables for the visible date range on mount and on navigation

**`DayDetail.tsx`** — expanded panel when a day is clicked:
- Lists all items for that day grouped by source
- Actions show status controls (same ActionCard from 1.2b)
- Activities, important dates, maintenance tasks show as read-only cards
- Slide-in panel or modal — your call on UX

### Data queries

For the visible month range, query:
```
actions: select * where due_date between [month_start] and [month_end]
activities: select * (filter recurring by day_of_week + one-time by date)
important_dates: select * where date between [month_start] and [month_end]
maintenance_tasks: select * where next_due between [month_start] and [month_end]
```

### Week view

Same data, different layout — 7 columns showing the current or selected week. Each column lists items vertically instead of showing dots. Denser than month view for short-term planning.
