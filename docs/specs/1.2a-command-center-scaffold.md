# Feature 1.2a: Command center scaffold

**Category:** `dashboards/command-center`

## Done when

- Next.js 14 App Router app exists at `dashboards/command-center/`
- Layout has nav with three tabs: Today (default), Upcoming, Thoughts
- Supabase client is configured in `src/lib/supabase.ts`
- QuickActionBar component renders at top (text input, placeholder functionality)
- StatsHeader component renders below nav (placeholder counts: open actions, overdue, thoughts this week, stale loops)
- Each tab route loads and shows placeholder content
- App runs locally with `npm run dev`
- `README.md` and `metadata.json` exist per CONTRIBUTING.md

## Context

This is the new primary dashboard вЂ” a cognitive load reducer, not a thought explorer. It answers: "What needs my attention?" and "What do I do next?" The existing `thought-explorer` stays deployed as a read-only review tool.

Clone `dashboards/data-browser/` as the starting skeleton.

### Stack

- Next.js 14 App Router
- Tailwind CSS
- `@supabase/supabase-js`
- Deploy target: Coolify

### Data access

- Supabase JS client with anon key (from environment variables)
- No RLS yet (single user) вЂ” add in Phase 2
- `.env.local.example` with `NEXT_PUBLIC_SUPABASE_URL` and `NEXT_PUBLIC_SUPABASE_ANON_KEY`

### Design intent

- **QuickActionBar:** Persistent at top of every view. Text input. Toggle between "capture as thought" (hits `ingest-thought` Edge Function) and "create as action" (inserts into `actions` table). For this scaffold session, build the UI but the submit handler can be a `console.log` placeholder.
- **StatsHeader:** Four boxes showing: open actions count, overdue count, thoughts captured this week, stale loops count. For this scaffold session, hardcode zeros вЂ” live queries come in 1.2b.
- **Nav tabs:** Simple top nav. Active tab highlighted. Links to `/`, `/upcoming`, `/thoughts`.

## Dependencies

None вЂ” this is a standalone scaffold. The view-specific specs (1.2b, 1.2c, 1.2d) build on top of this.
