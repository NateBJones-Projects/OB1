# Feature 1.1d-s2: Task engine — cron trigger evaluation

**Category:** Update `supabase/functions/scheduled-tasks-mcp/index.ts` (or wherever the task runner lives)

## Done when

- Task runner evaluates `cron_expression` on enabled tasks with `trigger_type = 'cron'`
- A `task-runner` Edge Function exists that can be called by `pg_cron` (or invoked via HTTP on a schedule)
- Runner checks which cron tasks are due in the current time window and executes them
- Manual trigger from 1.1d-s1 still works
- These tests work:
  ```
  Create a scheduled task: "hourly-test", trigger: cron, cron_expression: "0 * * * *",
  task_type: llm_prompt, delivery: mcp_response
  → Task registered

  [When the runner fires] → hourly-test executes if the current time matches its cron

  List scheduled tasks
  → Shows last_run_at updated for tasks that ran
  ```

## What to build

### Task runner Edge Function (`supabase/functions/task-runner/index.ts`)

Separate from the MCP tools function. Called by `pg_cron` on a regular interval (every 15 minutes recommended). On each invocation:

1. Query `scheduled_tasks` where `enabled = true` and `trigger_type = 'cron'`
2. For each task, evaluate whether `cron_expression` matches the current time window
3. For matched tasks, execute: gather → handler → delivery → log
4. Update `last_run_at` and `last_run_status` on the task

### Cron evaluation

Use a cron parsing library (e.g., `cron-parser` if available in Deno, or write a simple matcher for standard 5-field cron expressions). Compare the cron expression against the current time. Account for the runner's interval — if running every 15 minutes, a `0 7 * * *` task should fire when the runner runs at 7:00, not be missed because it runs at 7:15.

### pg_cron setup

Provide a SQL snippet to register the runner with `pg_cron`:
```sql
select cron.schedule(
  'task-runner',
  '*/15 * * * *',
  $$select net.http_post(url := 'https://[project-ref].supabase.co/functions/v1/task-runner', headers := '{"Authorization": "Bearer [service_role_key]"}'::jsonb);$$
);
```

(The actual URL and key come from environment variables — provide this as a template.)

## Dependencies

- 1.1d-s1 (task engine schema + MCP tools) must be built
