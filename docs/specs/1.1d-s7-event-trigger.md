# Feature 1.1d-s7: Task engine — event trigger

**Category:** Update task runner

## Done when

- Task runner evaluates `trigger_type = 'event'` tasks
- Checks Google Calendar for events within `event_lead_hours`
- Fires the task when a matching event is approaching
- This test works:
  ```
  Create a scheduled task: "meeting-prep", trigger: event, event_source: google_calendar,
  event_lead_hours: 2, task_type: llm_prompt, delivery: email
  → Task fires 2 hours before any calendar event
  ```

## What to build

### Event trigger evaluation

In the task runner's evaluation loop, for tasks with `trigger_type = 'event'`:
1. Query Google Calendar API for events in the next `event_lead_hours` hours
2. If events found and task hasn't already run for those events, trigger the task
3. Pass event details (title, attendees, time) into the gather phase as additional context

### Google Calendar integration

- Use Google Calendar API with service account credentials (from env vars)
- Query for events on the authenticated user's calendar
- Deduplicate: track which events have already triggered a task run (store event ID in `task_run_log.input_summary`)

## Dependencies

- 1.1d-s2 (cron triggers) — the runner infrastructure
- Google Calendar API credentials configured as environment variables
