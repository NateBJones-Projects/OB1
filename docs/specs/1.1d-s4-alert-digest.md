# Feature 1.1d-s4: Task engine — `alert_digest` task type

**Category:** Add handler to task runner

## Done when

- `alert_digest` task type handler is implemented in the task runner
- Handler scans configured tables for items due within threshold days
- Results grouped by urgency tier: due today, due in 3 days, due this week
- Formatted as a clean digest and delivered via configured channel
- This test works:
  ```
  Run alert-digest now
  → Returns grouped list of approaching due dates across actions, important dates, and maintenance tasks
  ```

## What to build

### Handler logic

1. Read `gather_config` for scan targets and thresholds:
   ```json
   {"scan_tables": ["actions", "important_dates", "maintenance_tasks"], "thresholds_days": [1, 3, 7]}
   ```
2. For each scan target, query items due within the max threshold:
   - `actions`: `status = 'open'` and `due_date` within N days
   - `important_dates`: `date` within N days
   - `maintenance_tasks`: `next_due` within N days
3. Group results by urgency tier (due today, due in 3 days, due this week)
4. Format as digest with section headers per tier
5. Deliver via `delivery_channel`
