# Feature 3.2: Stale loop detector (MCP tool)

**Category:** Add tool to `supabase/functions/open-brain-mcp/index.ts`

> This is the **on-demand MCP tool** mode. The scheduled mode (task type `stale_loop_scan`) is a separate spec at `docs/specs/1.1d-s5-stale-loop-scan.md`.

## Done when

- `detect_stale_loops` MCP tool exists in `open-brain-mcp/index.ts`
- Accepts `days` (default 14) and `limit` (default 20) params
- Returns stale actions (open > N days) with context on last related thought activity
- Returns unanswered questions (type=question, > N days old, no follow-up on same topic)
- Output format matches the example below
- This test works:
  ```
  Detect stale loops
  → Returns categorized list of stale actions and cold questions with last activity context
  ```

## What to build

### MCP tool definition

| Tool | Params | Description |
|------|--------|-------------|
| `detect_stale_loops` | `days?` (default 14), `limit?` (default 20) | Returns stale actions and unanswered questions with context on last related activity. |

### Logic

1. Query `actions` where `status = 'open'` and `created_at < now() - interval 'N days'`.
2. For each stale action, search `thoughts` for captures mentioning the same topics/people since the action was created. If none → truly stale. If found → thinking about it but loop unclosed.
3. Query `thoughts` where `metadata->>'type' = 'question'` and `created_at < now() - interval 'N days'` with no subsequent thought on the same topic.

### Output format

```
3 stale loops detected (>14 days):

1. [Action, 18 days old] "Schedule call with Integrity Air for HVAC ductwork"
   Last related capture: 12 days ago (mold remediation update)
   → Still active in your thinking but no action taken

2. [Question, 21 days old] "What's the Litmus digital twin answer for Matt?"
   No related captures since original
   → Completely cold — needs attention or cancellation

3. [Action, 16 days old] "Get Fabriq demo scheduled with Ingrid"
   Last related capture: 3 days ago (Fabriq NA planning note)
   → Active topic, but this specific action hasn't moved
```

## Dependencies

None — this is a standalone MCP tool addition. Quick win.
