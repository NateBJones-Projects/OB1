# DOK Pipeline

> Progressive knowledge enrichment from raw facts to Spiky Points of View.

## What It Does

This recipe implements a Depth of Knowledge (DOK) pipeline that progressively enriches raw facts into strategic insights and contrarian viewpoints. It automatically processes incoming thoughts through four levels:

1. **DOK1**: Raw facts and observations (base thoughts)
2. **DOK2**: Categorized knowledge clusters that group related facts
3. **DOK3**: Strategic insights that emerge from connecting knowledge across clusters
4. **DOK4**: Spiky Points of View (SPOVs) that challenge conventional wisdom

## Prerequisites

- Working Open Brain setup ([guide](../../docs/01-getting-started.md))
- Supabase project with the DOK pipeline schema (run `sql/06-dok-pipeline.sql`)
- OpenAI API key configured in Supabase secrets as `OPENAI_API_KEY`
- MCP access key configured as `MCP_ACCESS_KEY`

## Expected Outcome

When the pipeline runs successfully:

1. **DOK2 clusters** are created in the `dok_levels` table with `dok_level=2`, grouping related raw facts into coherent knowledge clusters

2. **DOK3 insights** are generated by analyzing patterns across DOK1 and DOK2 entries, stored with `dok_level=3`

3. **DOK4 SPOVs** are created as contrarian viewpoints that challenge conventional wisdom, stored with `dok_level=4` and fields like `spov_type`, `confidence`, and `validation_status`

4. The pipeline state tracks last processed timestamps in the `pipeline_state` table to avoid reprocessing the same content

5. You can query across all DOK levels using the `recall` tool to see the progressive enrichment from raw facts to strategic insights

## Troubleshooting

**Issue: Function deployment fails**
Solution: Check that your Supabase CLI is logged in and you have the correct project selected. Run `supabase status` to verify.

**Issue: "Invalid or missing access key" error**
Solution: Ensure the `MCP_ACCESS_KEY` secret is correctly set in your Supabase project and matches the key used in the curl commands.

**Issue: No documents being processed**
Solution: Check the pipeline status to see when it was last run. New documents must be added after the last processing timestamp to be picked up.

**Issue: Duplicate entries being created**
Solution: The pipeline includes deduplication at 0.95 similarity threshold. If duplicates appear, check the `match_dok_levels` function is working correctly.

**Issue: GPT API errors**
Solution: Verify your `OPENAI_API_KEY` secret is valid and has sufficient quota. Check the Supabase function logs for specific error messages.

**Issue: Slow processing**
Solution: The pipeline processes documents in batches of 15 per run to avoid timeout. For large backlogs, run the pipeline multiple times.
