# BrainLift Converter

## Problem

How to transform raw source materials into structured knowledge bases ("BrainLifts") optimized for both human comprehension and computational processing, with the ability to train LLMs on domain-specific insights.

## Trigger Conditions

- Convert source materials into BrainLift format
- Restructure data for LLM training
- Create knowledge bases with taxonomy development
- Generate spiky POVs from accumulated insights
- Analyze and validate source materials

## Process

### Phase 1: Restructuring
1. **Initial Assessment**
   - Review all provided source materials
   - Identify all Sources linked in materials
   - Map Sources to associated Insights
   - Identify key themes, experts, and relationships
   - Note information gaps requiring supplementation

2. **Taxonomy Development**
   - Create preliminary category structure
   - Validate categories against source material
   - Adjust based on information density and relevance

3. **Content Analysis**
   - Extract key concepts and relationships
   - Identify implicit connections between sources
   - Organize content within taxonomy

### Phase 2: Supplementation
1. **Expert Enhancement**
   - Research additional relevant experts
   - Verify current roles and contributions
   - Add complementary perspectives

2. **Source Enrichment**
   - Find supporting primary sources
   - Verify all new source links
   - Integrate new sources with existing content
   - Generate source summaries

3. **Insight Generation**
   - Analyze patterns across sources
   - Identify novel connections
   - Formulate contrarian viewpoints

4. **Spiky POV Creation**
   - Synthesize insights into unique, surprising/impactful, actionable Spiky Points of View
   - Classify as truths or myths
   - Apply confidence scoring

## Output

A structured BrainLift with hierarchical bullet point structure:

- [Topic Name]
  - Purpose
    - Reason: Clear statement of BrainLift creation purpose
    - How it can be used: Specific use cases and applications
    - In Scope: Detailed description with examples
    - Out of Scope: Clear boundaries with examples
  - Experts
    - [Expert Name]
      - Who: Current/prior roles with specific titles
      - Focus: Precise areas of expertise
      - Why Follow: Concrete contributions to field
      - Where: 1-3 verified content sources with working hyperlinks
  - DOK4 SPOV
    - Truths
      - Spiky POV Truth 1: Bold, surprising actionable truth
      - Spiky POV Truth 2: Bold, surprising actionable truth
      - Spiky POV Truth 3: Bold, surprising actionable truth
    - Myths
      - Spiky POV Myth 1: Commonly believed falsehood
      - Spiky POV Myth 2: Commonly believed falsehood
      - Spiky POV Myth 3: Commonly believed falsehood
  - DOK3 Insights (for each subcategory, 1-4 novel learnings)
    - Insight 1: Strategic insight from source summaries
    - Insight 2: Strategic insight from source summaries
    - Insight 3: Strategic insight from source summaries
  - DOK2 Knowledge Tree
    - [Specific Category Name] (5-8 total categories)
      - Summary: Concise overview
      - [Specific Subcategory Name] (2-4 per category)
        - Sources: (5-10 verified primary sources)
          - [Source 1 Name]
            - DOK1 Facts: Relevant facts extracted
            - Summary: Source 1 Summary
            - Link: Working hyperlink to source
          - [Source 2 Name]
            - DOK1 Facts: Relevant facts extracted
            - Summary: Source 2 Summary
            - Link: Working hyperlink to source

## Notes

- BrainLifts combine observations into actionable decisions
- Truths are surprising things that ARE true but most people wouldn't expect
- Myths are widely believed things that are NOT actually true
- SPOVs combine insights from 2+ domains and go beyond observation
- All hyperlinks must be functional and relevant
- Use precise, domain-specific terminology
- Create cross-references between sources, insights, and SPOVs
- Maintain consistent depth across categories
